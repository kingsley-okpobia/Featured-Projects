## What is PrivacyGuard UI?

PrivacyGuard UI is a **local-first PII redaction platform** that detects and removes personally identifiable information from CSV and JSON datasets вҖ” entirely on your machine, with zero cloud calls. Built for compliance officers, data scientists, and developers who need audit-ready data sanitisation without sending sensitive data to external APIs.

It wraps a purpose-built multilingual NER model in a clean, Lovable-designed interface and uses a two-pass architecture вҖ” vectorised regex for structured PII, neural NER for semantic entities вҖ” to achieve **sub-60-second processing for 10,000-row datasets on a standard CPU**.

### Two-pass scanning performance

| Dataset size | Regex pass | NER pass | Total (CPU) |
|---|---|---|---|
| 1,000 rows В· 5 cols | < 1s | ~3s | ~4s |
| 10,000 rows В· 5 cols | ~2s | ~40s | ~42s |
| 50,000 rows В· 5 cols | ~8s | ~3.5min | ~4min |

> Approximately **65% of cells** in typical business datasets are handled by the regex pass and never reach the NER model.

## Quick Start

### Prerequisites

- Python 3.9+
- [Conda](https://docs.conda.io) (recommended) or Python venv
- HuggingFace account + access token вҶ’ [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens)
- 8 GB RAM minimum В· No GPU required

### 1. Clone the repository

```bash
git clone https://github.com/kingsley-okpobia/Featured-Projects.git
cd Featured-Projects/privacyguard-backend
```

### 2. Create and activate environment

```bash
# Option A вҖ” conda (recommended)
conda create -n privacyguard python=3.11
conda activate privacyguard

# Option B вҖ” venv
python -m venv venv
# Windows:
venv\Scripts\activate
# macOS / Linux:
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Set your HuggingFace token

**Windows** вҖ” edit `run_app.bat` and replace `hf_xxxx`:
```bat
set HF_TOKEN=hf_your_actual_token_here
```

**macOS / Linux**:
```bash
export HF_TOKEN=hf_your_actual_token_here
```

### 5. Download the model (first run only)

```bash
python download_model.py
```

The model (`Babelscape/wikineural-multilingual-ner`, 278 MB) downloads to `./model`. Subsequent runs use the local cache.

### 6. Run the app

**Windows:**
```bat
run_app.bat
```

**macOS / Linux:**
```bash
streamlit run app.py --server.port 8501
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

## How It Works

### Redaction modes

**Redaction mode** replaces detected PII with typed tags:
```
"John Smith lives at 42 Oak Avenue, email john@example.com"
вҶ’ "[NAME] lives at [ADDRESS], email [EMAIL]"
```

**Synthetic mode** replaces PII with realistic fake equivalents:
```
"John Smith lives at 42 Oak Avenue, email john@example.com"
вҶ’ "Maria Garcia lives at 456 River Road, email user_7823@example.com"
```

### Detection sensitivity

| Setting | Threshold | Use case |
|---|---|---|
| Standard | 0.80 | High precision вҖ” clear, unambiguous PII only |
| High *(default)* | 0.60 | Balanced вҖ” recommended for most compliance use cases |
| Maximum | 0.50 | Maximum recall вҖ” review results before distributing |

### Column auto-detection

Columns are automatically pre-selected for scanning if their name contains: `name`, `email`, `phone`, `address`, `comment`, `note`, or `ip`. All selections can be manually overridden.

### Force-redact columns

Columns named `address`, `ip address`, `ip_address`, or similar are **force-redacted** вҖ” every non-empty cell is replaced without model inference, regardless of confidence.

## Compliance and Privacy

- **Zero egress architecture** вҖ” the application binds to `localhost` by default. No data reaches any external endpoint at any point.
- **No telemetry** вҖ” PrivacyGuard UI does not collect usage data, crash reports, or any form of analytics.
- **Audit trail** вҖ” every redaction session produces a downloadable audit CSV containing: column name, row index, original value preview (first 3 characters + `***`), entity type, confidence score, and action taken. This is designed to satisfy GDPR Article 30 record-keeping requirements.
- **Temporary files** вҖ” uploaded files are processed in memory and never written to disk outside the session.
- **Dependency SBOM** вҖ” all dependency versions are pinned in `requirements.txt`. Run `pip-audit` against this file to check for known vulnerabilities.

## Known Limitations

- **File size cap** вҖ” files over 50 MB are rejected at upload. For larger datasets, split into chunks before processing.
- **Nested JSON** вҖ” only one level of nesting is flattened. Deeply nested structures are not supported in v1.1.
- **Multilingual accuracy** вҖ” the wikineural model performs best on English, French, Spanish, and German. Accuracy on Arabic, Korean, and Hindi is lower; treat results as requiring human review.
- **Regex false positives** вҖ” the regex pre-filter has no semantic understanding. Version numbers matching date patterns (`12.5.2023`) or product codes matching phone patterns may be incorrectly flagged. The allowlist and diff view are the mitigation for this.
- **Streamlit CSS stability** вҖ” the UI uses `data-testid` attribute overrides for styling. Streamlit version updates may require CSS adjustments. Do not upgrade Streamlit without running a UI regression check.
